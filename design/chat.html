请解释一下 React useEffect 的依赖数组有哪些常见陷阱？

AI 助手 GPT-4o · 14:21

React useEffect 的依赖数组有几个常见陷阱：

  1. 对象/数组引用变化

    每次渲染都会创建新的对象引用，导致 useEffect 重复执行

    // 错误示例
useEffect(() => {
  console.log(user);
}, [user]); // user 每次都是新引用
  2. 函数引用问题

    内联函数每次渲染都会重新创建

  3. 循环依赖

    依赖数组中的值在 useEffect 内部被修改

解决方案：使用 useMemo 缓存对象，使用 useCallback 缓存函数。

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  // ✅ 使用 useMemo 缓存配置对象
  const fetchConfig = useMemo(() => ({
    headers: { 'Authorization': 'Bearer token' },
    timeout: 5000
  }), []);

  useEffect(() => {
    async function fetchUser() {
      try {
        setLoading(true);
        const response = await fetch(`/api/users/${userId}`, fetchConfig);
        const data = await response.json();
        setUser(data);
      } catch (error) {
        console.error('Failed to fetch user:', error);
      } finally {
        setLoading(false);
      }
    }
    fetchUser();
  }, [userId, fetchConfig]);

  • 使用 useMemo 缓存 fetchConfig
